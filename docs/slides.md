# Sales Copilot - короткая презентация

---

## 1. Задача

Менеджеру продаж нужен помощник до и после звонка:

- до звонка: быстро собрать контекст, цель, вопросы, возражения и договоренности;
- после звонка: понять, что получилось, что упущено и как улучшить следующий разговор;
- важно: вторая фаза должна проверять выполнение плана из первой фазы.

Демо-кейс: синтетический B2B SaaS-клиент Northstar Retail Cloud на этапе оценки 4-недельного пилота DealPilot AI.

---

## 2. Как решал

Собрал запускаемый CLI-прототип:

- Python без внешних пакетов;
- OpenAI Responses API;
- модель `gpt-5.4-mini`;
- два отдельных промпта: бриф и разбор;
- бриф фазы 1 передается в фазу 2 как обязательный контекст.

Поток данных:

`карточка + касания -> бриф -> транскрипт + бриф -> разбор + микро-урок`

---

## 3. Демо: вход -> выход

Фаза 1:

- вход: карточка клиента, письмо после вебинара, заметка менеджера после демо;
- выход: цель звонка, вопросы про HubSpot/KPI/CFO, вероятные возражения, список "не забыть".

Фаза 2:

- вход: транскрипт звонка и бриф фазы 1;
- выход: таблица выполнения плана, рекомендации, микро-урок по слабой фразе "давайте попробуем и посмотрим по ощущениям".

Скриншоты: `demo/screenshots/phase1.png`, `demo/screenshots/phase2.png`.

---

## 4. Выводы

Что получилось:

- обе фазы работают end-to-end на одном клиенте;
- разбор после звонка действительно использует бриф;
- рекомендации конкретные: proposal для CFO, владелец пилота, scope HubSpot, KPI threshold;
- микро-урок превращает слабую фразу в готовую B2B-формулировку.

Что улучшить:

- добавить JSON-схему и авто-проверки фактов;
- подключить RAG по sales playbook и базе возражений;
- встроить в CRM/Slack и call transcription pipeline;
- собирать аналитику повторяющихся ошибок по команде.
